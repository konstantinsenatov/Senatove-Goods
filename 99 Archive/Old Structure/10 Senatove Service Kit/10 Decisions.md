# Decisions

- First product is a **Premium Service WordPress Theme**.
- Product direction is **WordPress-first** over HTML/CSS-first.
- First release is not WooCommerce-first.
- Theme public name is **Senatove Service Theme**.
- Theme slug is `senatove-service-theme`.
- Companion plugin public name is **Senatove Sections Builder**.
- Plugin slug is `senatove-sections-builder`.
- Theme and plugin remain separate products in architecture.
- Plugin owns admin builder UX and section data structure.
- Theme owns frontend rendering and premium UI output.
- Builder is curated and controlled, not an open-ended generic page builder clone.
- Local app name is fixed as `senatove-service-kit`.
- Development environment is Local + WordPress + Cursor.
- Section-library-first approach is used for v1 scope control.
